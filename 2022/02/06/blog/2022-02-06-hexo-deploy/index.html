第一次使用 GitHub Actions 进行 hexo 自动化部署记录

什么是 GitHub Actions

GitHub Actions 是 GitHub 推出的持续集成服务。

GitHub 官方做了一个应用市场，在这里可以搜索到其他人提交的 actions。
每一个 acionts 都是一个独立的脚本。

GitHub Actions 的术语

  1. workflow（工作流）： 持续集成一次运行的过程就是一个 workflow
  2. job（任务）：一个 workflow 由多个 job 组成，就是一次持续集成的运行，可以执行多个任务。
  3. step（步骤）：每个 job 由多个 step 构成，由上到下依次完成。
  4. action（动作）： 每个 step 都是可以一次执行一个过多个 action

workflow 文件

此文件放置于项目根目录下的 .github/workflow目录。使用 yml 文件格式。如此项目中的 hexo-deploy.yml。一个代码仓库可以有多个 workflow 文件，且只要是在.github/workflow目录下的 .yml文件，GitHub 就会自动执行。

workflow 可以配置的项很多，详见官方文档

workflow 字段

  1. name

name 字段是 workflow 的名称。如果没有 name，就自动使用文件名作为 name

  1. on

on 字段是指定触发 workflow 的条件，通常是某个事件

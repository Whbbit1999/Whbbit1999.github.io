JavaScript中数组的操作

声明数组

在js中，可以使用字面量的方式来声明一个数组

也可以使用构造函数来声明一个数组

  1. 在构造函数中传入多个参数或者是包含字符串时，表示数组中的内容
  2. 传入单个的数字类型的参数时，表示这个数组的长度，数组的内容每一项都为empty
  3. 只传入一个负数时，会报错

数组的属性

length : 表示数组的长度

操作数组的方法

总览表

方法 作用 是否改变原数组 返回值
unshift() 前增 是 添加后数组的长度
push() 后增 是 添加后数组的长度
shift() 前删 是 被删除的元素
pop() 后删 是 被删除的元素
splice() 修改 是 被删除的元素数组
slice() 截取 否 截取到的元素数组
concat() 拼接 否 拼接后的数组
reverse() 反转 是 反转后的数组
sort() 排序 是 排序后的数组
toString() 数组转成字符串 否 转换后的字符串
join() 转成字符串 否 转换后的字符串

添加元素

会修改原数组

前增： unshift(元素1, 元素2, …,元素n)

返回添加后数组的长度

后增： push(元素1, 元素2, …,元素n)

返回数组添加后的长度

删除元素

会修改原数组

前删： shift()

返回被删除的元素,也就是原数组中的第一个元素

后删： pop()

返回被删除的元素,也就是原数组中最后一个元素

修改数组中的某一部分

splice(开始位置下标, 删除元素的长度,新增元素1, 新增元素2,…)

会修改原数组
返回被删除的元素数组
可以使用这个来对数组任意位置做增,删,改操作

截取数组中的某一部分

slice(开始位置的下标,结束位置的下标)

返回截取到的元素数组
不会影响原数组

拼接数组

concat(元素,元素,…)

在数组后面拼接新的元素
如果后面的元素是一个数组,那么就会将最外层数组展开拼接到后面
返回拼接后的新数组
不会影响原数组

数组排序

会修改原数组
返回排序后的数组
会影响原数组

倒序排列数组中的元素： reverse()

soft()

将数组中的元素按照字符编码进行从小到大的排序

想要让它按照数字大小进行排列,可以传递一个回调函数

数组转换为字符串

不会影响原数组
返回转换成的字符串

toString()

toString不是数组的方法,是Object的方法,数组继承于Object

  1. toString()可以将一个整数转换为一个指定进制的字符串

    传递的参数是要转换的进制(在2-36之间)

  2. toString()可以将数组转换成字符串

    join(‘连接符’)

    按照指定的连接符将数组转换成字符串

查看元素在数组中的位置

  1. indexOf(元素,从哪里开始向右找)
  2. lastIndexOf(元素,从哪里开始向左找)

遍历数组

  1. for in
  2. forEach()
    forEach()会遍历数组，循环体内没有返回值，forEach循环不会改变原数组的内容。
    forEach有三个参数，第一个参数是当前元素，第二个参数是当前元素的下标，第三个元素是当前元素所属数组。
  3. map()

    遍历数组，返回操作后的数组。不会改变原数组

map还可以直接把数组中的元素都转成字符串

  1. every()
  2. some()
  3. filter()
  4. reduce()

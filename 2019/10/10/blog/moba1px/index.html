如何处理移动端1px被渲染成2px的问题

在移动端项目中，安卓或者IOS等高分辨率屏幕会把1px的border渲染成2px来显示。如何解决这个问题呢？

局部处理

meta标签中的viewport属性，initial-scale 设置为1。rem 按照标准设计稿标准走，外加利用 transform 的scale(0.5) 缩小一倍即可

全局处理

meta标签中的viewport属性，initial-scale 设置为0.5。rem 按照标准设计稿标准走。

为什么修改viewport属性就可以解决呢？

这里就需要我们看一下meta标签各部分的意义了：
<meta name="viewport" content="width=device-width,initial-scale=1.0, maximum-scale=0.5, minimum-scale=0.5, user-scalable=no"/>

  • width : 设置网页的viewport宽度，为一个整数或者字符串 'device-width'
  • device-width : 设备宽度
  • height : 设置viewport的高度，一般设置了宽度后，高度可以自动解析，可以不用设置
  • initial-scale : 默认缩放比例，为一个数字，可以设置为小数
  • maximum-scale : 允许用户最大缩放比例，为一个数字，可以是小数
  • minimum-scale : 允许用户最小缩放比例，为一个数字，可以是小数
  • user-scalable : 是否允许手动缩放

为什么会产生这种问题？

UI 设计师设计的时候，画的 1px（真实像素）实际上是 0.5px(css) 的线或者边框。但是他不这么认为，他认为他画的就是 1px 的线，因为他画的稿的尺寸本身就是屏幕尺寸的 2 倍。假设手机视网膜屏的宽度是 320x480 宽，但实际尺寸是 640x960 宽，设计师设计图的时候一定是按照 640x960 设计的。但是前端工程师写代码的时候，所有 css 都是按照 320x480 写的，写 1px(css)，浏览器自动变成 2px（真实像素）。

那么前端工程师为什么不能直接写 0.5px(css) 呢？因为在老版本的系统里写 0.5px(css) 的话，会被浏览器解读为 0px(css)，就没有边框了。所以只能写成 1px(css)，实际在屏幕上显示出来就是设计师画的 1px（真实像素）的 2 倍那么宽，所以设计师会觉得这个线太粗了，和他的设计稿不一样。在新版的系统里，已经开始逐渐支持 0.5px(css) 这种写法。所以如果设计师在大图上设计了一个 1px（真实像素）的线的话，前端工程师直接除以 2，写 0.5px(css) 就好了。
